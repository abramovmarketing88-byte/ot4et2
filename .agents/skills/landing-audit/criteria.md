# Критерии аудита лендинга (15 модулей)

Оценивай каждый применимый модуль по шкале **1–10**.  
Для findings указывай: что не так → почему важно → что сделать.

Шкала: **1–3** критично плохо · **4–5** слабо · **6–7** приемлемо · **8–9** сильно · **10** эталон ниши.

---

## Модуль 1. Customer Research

Фундамент. Без понимания «для кого» нельзя судить дизайн и оффер.

### Изучить / восстановить

- JTBD (Jobs To Be Done)
- боли, страхи, желания, мотивация
- эмоциональные триггеры
- уровень знаний клиента
- уровень дохода / сегмент
- сценарии покупки, срочность проблемы
- кто принимает решение
- возражения перед покупкой

### Выход модуля

Чёткое понимание:

> Что человек чувствует **до** захода на сайт, что должен почувствовать **во время** просмотра и что должно подтолкнуть **оставить заявку**.

### Критерии (1–10)

| ID | Критерий |
|----|----------|
| 1.1 | ЦА явно считывается со страницы |
| 1.2 | Тон и визуал соответствуют сегменту (молодёжный бренд ≠ стройка) |
| 1.3 | Боли/желания отражены в копирайте |
| 1.4 | Возражения заранее закрыты |
| 1.5 | Срочность/контекст покупки учтены |
| 1.6 | Понятно, кто ЛПР и как с ним говорят |

## Модуль 2. Рынок и конкуренты

Нельзя сделать лучший лендинг, не понимая уровень рынка.

### Смотреть (если есть ссылки: 5–15 достаточно; 20–50 — идеал)

- лидеры отрасли, зарубежные, премиум, смежные ниши
- структура, офферы, фото, доверие
- повторяющиеся блоки, устаревшие приёмы

### Вывод

- что стандарт рынка
- чем выделиться
- что уже «замылило» взгляд

### Критерии

| ID | Критерий |
|----|----------|
| 2.1 | Страница не ниже визуального стандарта ниши |
| 2.2 | Оффер не сливается с типичными формулировками рынка |
| 2.3 | Есть хотя бы 1–2 отличия от шаблонных конкурентов |
| 2.4 | Нет устаревших клише ниши (если применимо) |
| 2.5 | Премиум-референсы учтены по уровню притязаний |

## Модуль 3. First Impression (2–3 секунды)

Пользователь ещё ничего не прочитал, но уже решил: дорого/дёшево, профессионалы ли, можно ли доверять, хочется ли читать дальше.

### Критерии

| ID | Критерий |
|----|----------|
| 3.1 | Визуальная привлекательность hero |
| 3.2 | Современность первого экрана |
| 3.3 | Ощущение качества / «не стыдно» |
| 3.4 | Эмоциональная реакция соответствует бренду |
| 3.5 | Желание остаться и скроллить |
| 3.6 | За 2–3 сек понятно: что предлагают и кому |
| 3.7 | CTA заметен без хаоса |
| 3.8 | Нет ощущения «шаблонного конструктора» |

**Порог:** модуль < 8 → обязателен фикс в текущем pass.

## Модуль 4. Visual Design Audit

Красивый дизайн — создание ощущения высокого качества, не «приятные цвета».

### 4A. Цветовая система

Гармония · контраст · фирменная палитра · эмоции · акценты · кнопки · фоны.

### 4B. Типографика

Плохие шрифты делают сайт дешёвым: выбор, размер, letter/line-spacing, читаемость, иерархия, единый стиль.  
Избегать дефолтных стеков (Inter, Roboto, Arial, system), если нет жёсткого брендбука.

### 4C. Композиция

Баланс · симметрия/асимметрия · композиционный центр · направление взгляда · группировка.

### 4D. Whitespace

Дорогой дизайн почти всегда с воздухом: отступы, отсутствие шума, комфорт, разделение блоков.

### 4E. Визуальная иерархия

Взглядом ясно: 1-е / 2-е / 3-е по важности. Хаотичный бег глаз = провал.

### 4F. Ритм страницы

Единая история: одинаковые расстояния/размеры, сетка, стиль блоков, повторяемость. Нарушенный ритм = дискомфорт без понимания «почему».

### 4G. Современность

Не «сайт 10 лет назад»: актуальный стиль, минимализм, аккуратность, без устаревшего декора.

### 4H. Премиальность

Даже недорогой бизнес может выглядеть дорого: фото, отступы, типографика, спокойные цвета, аккуратные тени, иконки, единый стиль.

### 4I. Фотографии

Разрешение · композиция · свет · реальные объекты · единая цветокоррекция · не дешёвый сток · эмоции людей.

### 4J. Иконки и графика

Единый стиль · толщина линий · размеры · нет случайных элементов.

### 4K. Микроанимации

Помогают и подчёркивают действия; не отвлекают. Цель: 2–3 осознанных motion, не шум.

### Критерии (сводка)

| ID | Критерий |
|----|----------|
| 4.1 | Цвет: гармония и контраст CTA/текста |
| 4.2 | Типографика: характер + читаемость + иерархия |
| 4.3 | Композиция и фокус |
| 4.4 | Достаточный воздух, нет визуального шума |
| 4.5 | Иерархия 1–2–3 считывается |
| 4.6 | Ритм и сетка стабильны |
| 4.7 | Современность 2020s+ |
| 4.8 | Премиальность ощущается |
| 4.9 | Фото продают, не портят |
| 4.10 | Иконки согласованы |
| 4.11 | Motion уместен |
| 4.12 | Нет «AI-дефолт» клише (фиолетовые градиенты, cream+terracotta serif, газетный broadsheet — если не осознанный бренд) |

## Модуль 5. UX

Красивый сайт не продаёт, если им неудобно пользоваться.

### Критерии

| ID | Критерий |
|----|----------|
| 5.1 | Сценарий пользователя до заявки ясен |
| 5.2 | Минимум лишних кликов/шагов |
| 5.3 | Интерфейс понятен без обучения |
| 5.4 | Формы: короткие, понятные лейблы, ошибки |
| 5.5 | Навигация / якорные переходы адекватны |
| 5.6 | Скорость принятия решения (не перегруз) |
| 5.7 | Мобильная версия: читаемость, тач-зоны, порядок |
| 5.8 | Кнопки доступны (размер, контраст, sticky CTA при длинной странице — по необходимости) |
| 5.9 | Путь до заявки виден с первого экрана и в конце |
| 5.10 | Нет ломающих UX эффектов (блокировка скролла, навязчивые попапы без причины) |

## Модуль 6. Копирайтинг и структура смыслов

Каждый экран отвечает на вопросы:

- Что это?
- Почему именно вы?
- Почему сейчас?
- Почему вам можно доверять?
- Как проходит работа?
- Сколько стоит?
- Какие гарантии?
- Что делать дальше?

Каждый блок логично продолжает предыдущий.

### Критерии

| ID | Критерий |
|----|----------|
| 6.1 | Hero отвечает «что / для кого / результат» |
| 6.2 | Блоки в логичном порядке истории |
| 6.3 | Нет воды и канцелярита |
| 6.4 | Конкретика: цифры, сроки, факты |
| 6.5 | Закрыты ключевые вопросы из списка выше |
| 6.6 | CTA-тексты = действие + ценность, не «Отправить» |
| 6.7 | FAQ / возражения (если ниша требует) |
| 6.8 | Единый tone of voice |

---

## Модуль 7. Оффер и позиционирование

### Критерии

| ID | Критерий |
|----|----------|
| 7.1 | Сильный заголовок (результат, не описание услуги) |
| 7.2 | Ценность понятна за 3–5 секунд чтения |
| 7.3 | Есть УТП / отличие от «как у всех» |
| 7.4 | Ясна проблема, которую решают |
| 7.5 | Оффер связан с сегментом ЦА |
| 7.6 | Подзаголовок усиливает, не повторяет заголовок |

Сильный паттерн: **результат + срок + снижение риска** (если уместно нише).

## Модуль 8. Психология принятия решений

Когнитивные триггеры (наличие **уместное**, не манипулятивный спам):

социальное доказательство · авторитет · доверие · дефицит · срочность · прозрачность · конкретика · гарантия · эффект владения · снижение риска · якорение цены · контраст · взаимность · последовательность · эмоциональные якоря

### Критерии

| ID | Критерий |
|----|----------|
| 8.1 | Соцдоказательства на виду |
| 8.2 | Снижение риска (гарантия, условия) |
| 8.3 | Срочность/дефицит честные, не фейк |
| 8.4 | Конкретика вместо абстракций |
| 8.5 | Эмоциональные якоря совпадают с ЦА |
| 8.6 | Цена/пакеты якорятся и сравниваются ясно |
| 8.7 | Триггеры не противоречат друг другу и не выглядят дёшево |

## Модуль 9. Доверие

Пользователь должен почувствовать: *«Этой компании можно доверить деньги.»*

Проверить наличие (по нише — не всё обязательно, но критичные дыры = низкий балл):

руководитель · команда · фото сотрудников · сертификаты · лицензии · договоры · гарантии · отзывы · видео · кейсы · реальные объекты · документы · реквизиты · СМИ · партнёры

### Критерии

| ID | Критерий |
|----|----------|
| 9.1 | Живые доказательства, не сток-«команда» |
| 9.2 | Отзывы/кейсы конкретны |
| 9.3 | Юр./контактная прозрачность |
| 9.4 | Гарантии сформулированы ясно |
| 9.5 | Нет красных флагов (анонимность, давление, несостыковки) |
| 9.6 | Proof рядом с местами сомнения (цена, форма) |

## Модуль 10. Анализ конверсии (воронка)

Модель пути (пример):

```
1000 посетителей
→ 700 просмотрели первый экран
→ 450 дошли до преимуществ
→ 180 посмотрели кейсы
→ 70 нажали кнопку
→ 18 отправили заявку
```

Без аналитики — **качественная** карта: где внимание должно теряться и почему (гипотезы).

### Критерии

| ID | Критерий |
|----|----------|
| 10.1 | Hero удерживает и ведёт дальше |
| 10.2 | Нет «стены текста» до ценности |
| 10.3 | CTA повторяются в точках готовности |
| 10.4 | Слабые/пустые блоки не убивают скролл |
| 10.5 | Форма не перегружена |
| 10.6 | Финальный блок закрывает сделку |

## Модуль 11. Аналитика поведения

Инструменты: карты кликов/скролла · вебвизор · теплокарты · аналитика форм · глубина · время · возвраты · источники.

**Без доступа:** статус `N/A`, рекомендации что включить. Не ставь выдуманный балл выше 0 вклада в итог.

### Если данные есть — критерии

| ID | Критерий |
|----|----------|
| 11.1 | Скролл до ключевых блоков достаточен |
| 11.2 | Клики в CTA, не в мёртвые зоны |
| 11.3 | Форма: где бросают |
| 11.4 | Расхождение desktop/mobile |
| 11.5 | Источники с худшей конверсией учтены |

## Модуль 13. План улучшений

Не балл «красоты», а **качество плана**: полнота, приоритеты, выполнимость.

Каждая проблема: описание · причина · влияние на конверсию · сложность · эффект · приоритет внедрения · действие.

---

## Модуль 14. A/B гипотезы

После фиксов — список гипотез (не имитация результатов тестов):

первый экран · оффер · фото · кнопки · формы · заголовки · порядок блоков · отзывы · CTA

Формат гипотезы: *Если [изменение], то [метрика] вырастет, потому что [причина].*

---

## Модуль 15. Непрерывная оптимизация

Лендинг не «закончен навсегда». Рекомендовать мониторинг:

конверсия · стоимость заявки · качество лидов · время на сайте · глубина · поведение

И следующий цикл улучшений по данным.

---

## Быстрый чек «четыре уровня»

Перед выставлением итога пробегись:

1. **Эстетика** — хочется ли смотреть? Премиально ли?
2. **Удобство** — легко ли оставить заявку с телефона?
3. **Психология** — снята ли тревога? Есть ли доверие?
4. **Маркетинг** — ясен ли оффер и путь к действию?
